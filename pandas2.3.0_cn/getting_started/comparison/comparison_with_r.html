与 R / R 库的比较#

由于 pandas 旨在提供人们使用 R 进行的许多数据操作和分析功能，因此本页面旨在更详细地介绍 R 语言及其许多第三方库与 pandas 的关系。在与 R 和 CRAN 库的比较中，我们关注以下几点：

  • 功能性/灵活性：每个工具能做什么/不能做什么

  • 性能：操作的速度。最好有具体数字/基准

  • 易用性：哪个工具更容易/更难使用（鉴于并排的代码比较，您可能需要自行判断）

本页面还为这些 R 包的用户提供了一个翻译指南。

match / %in%#

在 R 中选择数据的常用方法是使用 %in%，它使用函数 match 定义。%in% 运算符用于返回一个逻辑向量，指示是否存在匹配

s <- 0:4
s %in% c(2,4)

isin() 方法与 R 的 %in% 运算符相似

In [12]: s = pd.Series(np.arange(5), dtype=np.float32)

In [13]: s.isin([2, 4])
Out[13]: 
0    False
1    False
2     True
3    False
4     True
dtype: bool

match 函数返回一个向量，其中包含其第一个参数在其第二个参数中的匹配位置

s <- 0:4
match(s, c(2,4))

有关更多详细信息和示例，请参阅 重塑文档。

tapply#

tapply 类似于 aggregate，但数据可以是不规则数组，因为子类大小可能不规则。使用名为 baseball 的 data.frame，并根据数组 team 检索信息

baseball <-
  data.frame(team = gl(5, 5,
             labels = paste("Team", LETTERS[1:5])),
             player = sample(letters, 25),
             batting.average = runif(25, .200, .400))

tapply(baseball$batting.average, baseball.example$team,
       max)

在 pandas 中，我们可以使用 pivot_table() 方法来处理这个问题

subset#

df <- data.frame(a=rnorm(10), b=rnorm(10))
subset(df, a <= b)
df[df$a <= df$b,]  # note the comma

在 R 中，acast 是一个使用名为 df 的 data.frame 转换成更高维数组的表达式

df <- data.frame(

  z = runif(12, 1.7, 20.7),

mdf <- melt(df, id=c("month", "week"))
acast(mdf, week ~ month ~ variable, mean)

factor#

pandas 有一个用于分类数据的数据类型。

cut(c(1,2,3,4,5,6), 3)
factor(c(1,2,3,2,2,3))

在 pandas 中，这通过 pd.cut 和 astype("category") 实现

In [41]: pd.cut(pd.Series([1, 2, 3, 4, 5, 6]), 3)
Out[41]: 
0    (0.995, 2.667]
1    (0.995, 2.667]
2    (2.667, 4.333]
3    (2.667, 4.333]
4      (4.333, 6.0]
5      (4.333, 6.0]
dtype: category
Categories (3, interval[float64, right]): [(0.995, 2.667] < (2.667, 4.333] < (4.333, 6.0]]

In [42]: pd.Series([1, 2, 3, 2, 2, 3]).astype("category")
Out[42]: 
0    1
1    2
2    3
3    2
4    2
5    3
dtype: category
Categories (3, int64): [1, 2, 3]

有关更多详细信息和示例，请参阅 分类介绍 和 API 文档。另有关于 与 R 的 factor 的差异 的文档。
