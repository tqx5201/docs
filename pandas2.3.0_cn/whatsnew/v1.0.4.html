1.0.4 版新特性 (2020年5月28日)#

以下是 pandas 1.0.4 版中的更改。有关包括其他 pandas 版本在内的完整更新日志，请参阅发布说明。

已修复的回归问题#

  • 修复了在 pandas.options.mode.use_inf_as_na 设置为 True 时，Series.isna() 和 DataFrame.isna() 会对分类数据类型 (categorical dtype) 引发异常的回归问题 (GH 33594)

  • 修复了 DataFrameGroupBy.first()、SeriesGroupBy.first()、DataFrameGroupBy.last() 和 SeriesGroupBy.last() 中，`None` 值在对象数据类型 (object dtype) 中未被保留的回归问题 (GH 32800)

  • 修复了在 DataFrame 规约操作中使用 numeric_only=True 和扩展数组 (ExtensionArrays) 时的回归问题 (GH 33256)。

  • 修复了 memory_usage(deep=True) 对于对象数据类型 (object dtype) 的性能回归问题 (GH 33012)

  • 修复了 Categorical.replace() 在新值与替换值相等时，会用 NaN 进行替换的回归问题 (GH 33288)

  • 修复了有序 Categorical（仅包含 NaN 值）在取最小值或最大值时会引发异常而非返回 NaN 的回归问题 (GH 33450)

  • 修复了 DataFrameGroupBy.agg() 使用字典输入时丢失 ExtensionArray 数据类型 (dtypes) 的回归问题 (GH 32194)

  • 修复了通过 xarray 的 CFTimeIndex（Index 的子类）使用“最近”方法进行索引的能力未被保留的问题 (pydata/xarray#3751, GH 32905)。

  • 修复了 DataFrame.describe() 引发 TypeError: unhashable type: 'dict' 的回归问题 (GH 32409)

  • 修复了 DataFrame.replace() 在 to_replace 中的项不在值中时，将列转换为 object 数据类型 (dtype) 的回归问题 (GH 32988)

  • 修复了 Series.groupby() 在按 PeriodIndex 级别分组时会引发 ValueError 的回归问题 (GH 34010)

  • 修复了 DataFrameGroupBy.rolling.apply() 和 SeriesGroupBy.rolling.apply() 忽略 `args` 和 `kwargs` 参数的回归问题 (GH 33433)

  • 修复了在无序 Categorical 上使用 np.min 或 np.max 时错误消息的回归问题 (GH 33115)

  • 修复了 DataFrame.loc() 和 Series.loc() 在提供 datetime64[ns, tz] 值时会引发错误的问题 (GH 32395)

错误修复#

  • 修复了 SeriesGroupBy.first()、SeriesGroupBy.last()、SeriesGroupBy.min() 和 SeriesGroupBy.max() 应用于可空布尔值 (nullable Booleans) 时返回浮点数的错误 (GH 33071)

  • 修复了 Rolling.min() 和 Rolling.max() 中的错误：在使用固定窗口时多次调用后内存占用持续增长 (GH 30726)

  • 修复了 to_parquet() 在使用无效凭据写入私有 S3 存储桶时未引发 PermissionError 的错误 (GH 27679)

  • 修复了 to_csv() 在写入无效 S3 存储桶时静默失败的错误 (GH 32486)

  • 修复了 read_parquet() 在传入 S3 目录路径时会引发 FileNotFoundError 的错误 (GH 26388)

  • 修复了 to_parquet() 在将分区 Parquet 文件写入 S3 时会引发 AttributeError 的错误 (GH 27596)

  • 修复了 DataFrameGroupBy.quantile() 和 SeriesGroupBy.quantile() 在 by 轴包含 NaN 时导致分位数偏移的错误 (GH 33200, GH 33569)
