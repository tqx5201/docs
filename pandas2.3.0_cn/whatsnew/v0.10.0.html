版本 0.10.0（2012 年 12 月 17 日）#

这是 0.9.1 的一个主要版本，包含许多新功能和增强功能，以及大量的错误修复。此外还有一些重要的 API 更改，pandas 的长期用户应密切关注。

文件解析新功能#

分隔文件解析引擎（read_csv 和 read_table 的核心）已从头重写，现在解析时内存使用量极少，并且在大多数用例中速度提升 40% 或更多（在某些情况下更快）。

此外还有许多新功能：

  • 通过 encoding 选项大大改进了 Unicode 处理。

  • 列过滤（usecols）

  • 数据类型指定（dtype 参数）

  • 能够指定被识别为 True/False 的字符串

  • 能够生成 NumPy 记录数组（as_recarray）

  • 高性能 delim_whitespace 选项

  • 十进制格式（例如欧洲格式）指定

  • 更简单的 CSV 方言选项：escapechar、lineterminator、quotechar 等。

  • 更健壮地处理实际中遇到的多种异常文件类型

API 更改#

弃用 DataFrame BINOP TimeSeries 特殊情况行为

DataFrame 和 Series 之间的二元运算的默认行为一直是基于 DataFrame 的列对齐并向下广播到行，**除了** DataFrame 包含时间序列的特殊情况。由于现在每个二元运算符都有方法允许您指定如何广播，我们正在逐步淘汰这种特殊情况（Python 之禅：*特殊情况不足以打破规则*）。具体如下：

  • 带有 inplace 选项的方法现在都返回 None，而不是调用对象。例如，像 df = df.fillna(0, inplace=True) 这样的代码可能停止工作。要修复，只需删除不必要的变量赋值。

  • pandas.merge 默认不再对分组键进行排序（sort=False）。这是出于性能考虑：分组键排序通常是计算中开销较大的部分之一，而且通常是不必要的。

  • 没有标题的文件默认列名已更改为从 0 到 N - 1 的整数。这是为了与未指定列的 DataFrame 构造函数保持一致。v0.9.0 的行为（名称为 X0、X1 等）可以通过指定 prefix='X' 来重现。

Out[6]:

0      1   True  2
1      3  False  4

  • 表写入性能改进

  • 支持任意索引维度

  • SparseSeries 现在有一个 density 属性（GH 2384）

  • 启用 Series.str.strip/lstrip/rstrip 方法接受输入参数以去除任意字符（GH 2411）

  • 在 melt 中实现 value_vars 以限制值在特定列中，并将 melt 添加到 pandas 命名空间（GH 2412）

错误修复

  • 添加了指定 where 条件的 Term 方法（GH 1996）。

  • del store['df'] 现在调用 store.remove('df') 来删除存储

  • 删除连续行比以前快得多

  • 在表创建时可以指定 min_itemsize 参数，以强制索引列的最小大小（以前的实现会根据第一次追加来设置列大小）

  • 通过 create_table_index 支持索引（需要 PyTables >= 2.3）（GH 698）。

  • 如果表没有首先通过 put 创建，则在存储上追加会失败

  • 修复了加载序列化 DataFrame 后属性缺失的问题 (GH2431)

  • 对选择和删除进行了小幅更改：仅当同时提供了 where 条件（且不为 None）时才需要表

兼容性

HDFStore 的 0.10 版本在读取使用 pandas 早期版本创建的表时向后兼容，但是，不支持使用先前（未文档化）方法查询的术语。您必须读取整个文件并使用新格式将其写出，才能利用这些更新。
