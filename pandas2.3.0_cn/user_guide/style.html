表格可视化#

本节演示如何使用 Styler 类进行表格数据的可视化。有关图表可视化的信息，请参阅 图表可视化。本文档以 Jupyter Notebook 形式编写，可在此处查看或下载。

Styler 对象和显示自定义#

样式和输出显示自定义应在 DataFrame 中的数据处理之后进行。如果 DataFrame 发生进一步更改，Styler 不会动态更新。DataFrame.style 属性是一个返回 Styler 对象的属性。它定义了 _repr_html_ 方法，因此它会在 Jupyter Notebook 中自动渲染。

Styler 主要设计用于小数据，但也可用于大数据，目前能够输出以下格式：

  • HTML

  • LaTeX

  • 字符串（以及扩展的 CSV）

  • Excel

  • （JSON 目前不可用）

其中前三种格式具有设计用于格式化和自定义输出的显示自定义方法。这些方法包括：

  • 使用 .format() 和 .format_index() 格式化值、索引和列标题，

  • 使用 .relabel_index() 重命名索引或列标题标签，

  • 使用 .hide() 隐藏某些列、索引和/或列标题，或索引名称，

  • 使用 .concat() 拼接相似的 DataFrame。

格式化显示#

格式化值#

Styler 区分显示值和实际值，包括数据值以及索引或列标题。为了控制显示值，每个单元格中的文本都以字符串形式打印，我们可以使用 .format() 和 .format_index() 方法根据格式规范字符串或接受单个值并返回字符串的可调用对象来操作它。可以为整个表格或索引，或为单个列或 MultiIndex 级别定义此操作。我们还可以覆盖索引名称。

此外，格式化函数还具有一个 precision 参数，专门用于帮助格式化浮点数，以及 decimal 和 thousands 分隔符以支持其他区域设置，一个 na_rep 参数用于显示缺失数据，以及 escape 和 hyperlinks 参数用于帮助显示安全 HTML 或安全 LaTeX。默认格式化程序配置为采用 pandas 的全局选项，例如 styler.format.precision 选项，可通过 with pd.option_context('format.precision', 2): 控制。

[2]:

import pandas as pd
import numpy as np
import matplotlib as mpl

df = pd.DataFrame({
    "strings": ["Adam", "Mike"],
    "ints": [1, 3],
    "floats": [1.123, 1000.23]
})
df.style \
  .format(precision=3, thousands=".", decimal=",") \
  .format_index(str.upper, axis=1) \
  .relabel_index(["row 1", "row 2"], axis=0)

隐藏数据#

索引和列标题可以完全隐藏，也可以选择性地排除某些行或列。这两种选项都使用相同的方法来实现。

通过不带任何参数调用 .hide() 可以隐藏索引，这在索引是基于整数时可能很有用。同样，通过不带任何进一步参数调用 .hide(axis=”columns”) 可以隐藏列标题。

可以通过调用相同的 .hide() 方法并将行/列标签、类似列表的对象或行/列标签的切片传递给 subset 参数来隐藏特定的行或列。

隐藏不会改变 CSS 类的整数排列，例如，隐藏 DataFrame 的前两列意味着列类索引仍将从 col2 开始，因为 col0 和 col1 只是被忽略了。

[5]:

df = pd.DataFrame(np.random.randn(5, 5))
df.style \
  .hide(subset=[0, 2, 4], axis=0) \
  .hide(subset=[0, 2, 4], axis=1)

Styler 对象和 HTML#

Styler 最初是为了支持广泛的 HTML 格式选项而构建的。其 HTML 输出会创建 HTML <table> 并利用 CSS 样式语言来操作许多参数，包括颜色、字体、边框、背景等。有关 HTML 表格样式的更多信息，请参阅此处。这提供了很大的开箱即用灵活性，甚至使网页开发者能够将 DataFrames 集成到他们现有的用户界面设计中。

添加样式的方法#

为 Styler 添加自定义 CSS 样式有3 种主要方法：

  • 使用 .set_table_styles() 控制表格的更广泛区域，并指定内部 CSS。虽然表格样式允许灵活添加 CSS 选择器和属性来控制表格的所有各个部分，但对于单个单元格的规范来说却显得笨拙。另请注意，表格样式无法导出到 Excel。

  • 使用 .set_td_classes() 直接将外部 CSS 类链接到您的数据单元格，或链接由 .set_table_styles() 创建的内部 CSS 类。请参阅此处。这些方法不能用于列标题行或索引，也无法导出到 Excel。

  • 使用 .apply() 和 .map() 函数为特定数据单元格添加直接内部 CSS。请参阅此处。自 v1.4.0 起，还有直接作用于列标题行或索引的方法；.apply_index() 和 .map_index()。请注意，只有这些方法添加的样式才能导出到 Excel。这些方法的工作方式与 DataFrame.apply() 和 DataFrame.map() 类似。

表格样式#

表格样式足够灵活，可以控制表格的所有单独部分，包括列标题和索引。但是，对于单个数据单元格或任何类型的条件格式，它们可能很难输入，因此我们建议将表格样式用于广泛的样式设置，例如一次性对整个行或列进行样式设置。

表格样式还用于控制可以一次性应用于整个表格的功能，例如创建通用的悬停功能。:hover 伪选择器以及其他伪选择器只能以这种方式使用。

为了复制 CSS 选择器和属性（属性值对）的常规格式，例如

tr:hover {
  background-color: #ffff99;
}

将样式传递给 .set_table_styles() 的必要格式是字典列表，每个字典都包含一个 CSS 选择器标签和 CSS 属性。属性可以是 2 元组列表，也可以是常规 CSS 字符串，例如

[13]:

cell_hover = {  # for row hover use <tr> instead of <td>
    'selector': 'td:hover',
    'props': [('background-color', '#ffffb3')]
}
index_names = {
    'selector': '.index_name',
    'props': 'font-style: italic; color: darkgrey; font-weight:normal;'
}
headers = {
    'selector': 'th:not(.index_name)',
    'props': 'background-color: #000066; color: white;'
}
s.set_table_styles([cell_hover, index_names, headers])

设置类并链接到外部 CSS#

如果您已经设计了一个网站，那么您很可能已经有一个外部 CSS 文件来控制其中表格和单元格对象的样式。您可能希望使用这些原生文件，而不是在 Python 中复制所有 CSS（并复制任何维护工作）。

表格属性#

使用 .set_table_attributes() 可以非常轻松地向主 <table> 添加 class。此方法还可以附加内联样式——在 CSS 层级 中阅读更多信息。

数据单元格 CSS 类#

版本 1.2.0 新增

.set_td_classes() 方法接受一个 DataFrame，其索引和列与底层 Styler 的 DataFrame 匹配。该 DataFrame 将包含字符串作为 CSS 类，以添加到各个数据单元格中：即 <table> 的 <td> 元素。我们不会使用外部 CSS，而是将在内部创建类并将其添加到表格样式中。我们将把添加边框的工作留到工具提示和标题部分。

s.set_table_styles([  # create internal CSS classes
    {'selector': '.true', 'props': 'background-color: #e6ffe6;'},
    {'selector': '.false', 'props': 'background-color: #ffe6e6;'},
], overwrite=False)
cell_color = pd.DataFrame([['true ', 'false ', 'true ', 'false '],

Styler 函数#

作用于数据#

我们使用以下方法来传递您的样式函数。这两种方法都接受一个函数（和一些其他关键字参数），并以特定方式将其应用于 DataFrame，从而渲染 CSS 样式。

  • .map() (逐元素)：接受一个函数，该函数接受单个值并返回一个包含 CSS 属性-值对的字符串。

  • .apply()（按列/行/表）：接受一个函数，该函数接受 Series 或 DataFrame 并返回具有相同形状的 Series、DataFrame 或 numpy 数组，其中每个元素都是一个包含 CSS 属性-值对的字符串。此方法一次传递 DataFrame 的每一列或每一行，或一次传递整个表，具体取决于 axis 关键字参数。对于按列使用 axis=0，按行使用 axis=1，对于整个表一次性使用 axis=None。

优化#

通常，对于较小的表格和大多数情况，渲染的 HTML 不需要优化，我们也不太建议这样做。在以下两种情况下值得考虑：

  • 如果您正在渲染和样式一个非常大的 HTML 表格，某些浏览器可能会出现性能问题。

  • 如果您使用 Styler 动态创建在线用户界面的一部分并希望提高网络性能。

这里我们推荐以下实施步骤：

1. 移除 UUID 和 cell_ids#

忽略 uuid 并将 cell_ids 设置为 False。这将防止不必要的 HTML。

这是次优的

[41]:

df4 = pd.DataFrame([[1,2],[3,4]])
s4 = df4.style

这更好

[42]:

from pandas.io.formats.style import Styler
s4 = Styler(df4, uuid_len=0, cell_ids=False)

2. 使用表格样式#

尽可能使用表格样式（例如，一次性应用于所有单元格或行或列），因为 CSS 几乎总是比其他格式更高效。

这是次优的

[43]:

df4.style.map(lambda x: props, subset=[1])

导出到 Excel#

（自版本 0.20.0 起）支持使用 OpenPyXL 或 XlsxWriter 引擎将样式化 DataFrames 导出到 Excel 工作表。处理的 CSS2.2 属性包括：

  • background-color（背景颜色）

  • border-style 属性（边框样式）

  • border-width 属性（边框宽度）

  • border-color 属性（边框颜色）

  • color（颜色）

  • font-family（字体族）

  • font-style（字体样式）

  • font-weight（字体粗细）

  • text-align（文本对齐）

  • text-decoration（文本装饰）

  • vertical-align（垂直对齐）

  • white-space: nowrap（空白：不换行）

  • 支持简写和侧边特定边框属性（例如 border-style 和 border-left-style），以及所有侧边（border: 1px solid green）或指定侧边（border-left: 1px solid green）的 border 简写。使用 border 简写将覆盖其之前设置的任何边框属性（有关详细信息，请参阅 CSS 工作组）。

  • 目前仅支持 CSS2 命名颜色和 #rgb 或 #rrggbb 形式的十六进制颜色。

  • 以下伪 CSS 属性也可用于设置 Excel 特定样式属性：

    • number-format（数字格式）

    • border-style（用于 Excel 特定样式：“hair”、“mediumDashDot”、“dashDotDot”、“mediumDashDotDot”、“dashDot”、“slantDashDot”或“mediumDashed”）

表格级别样式和数据单元格 CSS 类不包含在导出到 Excel 中：单个单元格必须通过 Styler.apply 和/或 Styler.map 方法映射其属性。

层叠样式表（CSS）语言旨在影响浏览器如何渲染 HTML 元素，它有自己的特殊性。它从不报告错误：它只是默默地忽略它们，并且不会按照您的意图渲染您的对象，因此有时可能会令人沮丧。这里简要介绍 Styler 如何创建 HTML 并与 CSS 交互，并提供避免常见陷阱的建议。

CSS 类和 ID#

附加到每个单元格的 CSS class 的精确结构如下。

  • 包含索引和列名称的单元格包括 index_name 和 level<k>，其中 k 是其在 MultiIndex 中的级别

  • 索引标签单元格包括

    • 行标题

    • level<k>，其中 k 是 MultiIndex 中的级别

    • row<m>，其中 m 是行的数字位置

  • 列标签单元格包括

    • 列标题

    • level<k>，其中 k 是 MultiIndex 中的级别

    • col<n>，其中 n 是列的数字位置

  • 数据单元格包括

    • 数据

    • row<m>，其中 m 是单元格的数字位置。

    • col<n>，其中 n 是单元格的数字位置。

  • 空白单元格包含 blank

  • 修剪过的单元格包括 col_trim 或 row_trim

id 的结构是 T_uuid_level<k>_row<m>_col<n>，其中 level<k> 仅用于标题，并且标题将仅包含 row<m> 或 col<n>，取所需者。默认情况下，我们还在每个行/列标识符前添加了一个对于每个 DataFrame 唯一的 UUID，这样同一个笔记本或页面中的样式就不会与另一个样式冲突。您可以在优化中阅读更多关于 UUID 使用的信息。

我们可以通过调用 .to_html() 方法来查看 HTML 示例。

CSS 层级#

示例表明，当 CSS 样式重叠时，HTML 渲染中最后出现的样式优先。因此，以下代码会产生不同的结果：

[72]:

df4 = pd.DataFrame([['text']])
df4.style.map(lambda x: 'color:green;')\
         .map(lambda x: 'color:red;')

[72]:
  0
0 文本 
[73]:

df4.style.map(lambda x: 'color:red;')\
         .map(lambda x: 'color:green;')

[73]:
  0
0 文本

这仅适用于层级或重要性相同的 CSS 规则。您可以阅读更多关于CSS 特异性的信息，但为了我们的目的，总结关键点就足够了：

每个 HTML 元素的 CSS 重要性分数通过从零开始并添加以下内容得出：

  • 内联样式属性得 1000 分

  • 每个 ID 得 100 分

  • 每个属性、类或伪类得 10 分

  • 每个元素名称或伪元素得 1 分

我们用这个来描述以下配置的动作

[74]:

df4.style.set_uuid('a_')\
         .set_table_styles([{'selector': 'td', 'props': 'color:red;'}])\
         .map(lambda x: 'color:green;')

[74]:
  0
0 文本

此文本为红色，因为生成的选择器 #T_a_ td 的值为 101（ID 加元素），而 #T_a_row0_col0 仅为 100（ID），因此即使在 HTML 中它在前面之后，也被认为是次要的。

df4.style.set_uuid('b_')\
         .set_table_styles([{'selector': 'td', 'props': 'color:red;'},

         .map(lambda x: 'color:green;')\
         .set_td_classes(pd.DataFrame([['cls-1']]))

在上述情况下，文本是蓝色的，因为选择器 #T_b_ .cls-1 的值为 110（ID 加类），它具有优先权。

子类化#

如果默认模板不能完全满足您的需求，您可以子类化 Styler 并扩展或覆盖模板。我们将展示一个扩展默认模板以在每个表格前插入自定义标题的示例。

[78]:

from jinja2 import Environment, ChoiceLoader, FileSystemLoader
from IPython.display import HTML
from pandas.io.formats.style import Styler

我们将使用以下模板

[79]:

with open("templates/myhtml.tpl") as f:
    print(f.read())

{% extends "html_table.tpl" %}
{% block table %}
<h1>{{ table_title|default("My Table") }}</h1>
{{ super() }}
{% endblock table %}

现在我们已经创建了一个模板，我们需要设置一个 Styler 的子类，它了解这个模板。Styler

[80]:

class MyStyler(Styler):
    env = Environment(
        loader=ChoiceLoader([
            FileSystemLoader("templates"),  # contains ours
            Styler.loader,  # the default
        ])
    )
    template_html_table = env.get_template("myhtml.tpl")

请注意，我们将原始加载器包含在我们的环境加载器中。这是因为我们扩展了原始模板，所以 Jinja 环境需要能够找到它。
