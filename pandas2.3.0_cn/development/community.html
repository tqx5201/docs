贡献者社区#

pandas 是一个社区驱动的开源项目，由大量贡献者和一小部分维护者共同开发。pandas 领导层已坚定致力于创建一个开放、包容和积极的社区。请阅读 pandas 的行为准则，以获取如何与他人互动以促进社区繁荣发展的指导。

我们提供多种会议和沟通渠道，以便在 pandas 社区内分享知识并与他人联系。

社区会议#

pandas 社区会议是项目维护者的定期同步会议，向社区开放。欢迎所有人参加并为讨论做出贡献。

会议在每月第二个和第四个星期三的协调世界时 18:00 举行。

过去的会议纪要可在此 Google 文档中查阅。

新贡献者会议#

在每月的第三个星期三，我们会举行会议，欢迎和支持社区中的新贡献者。

👋 欢迎大家参加
💬 欢迎所有人发言（将自己添加到 hackMD 议程中）
👀 欢迎旁听

参会者包括新旧贡献者以及一些维护者。我们的目标是解答关于入门的问题，尽可能帮助进行中的工作，同时互相认识并分享我们的学习和经验。

下次会议议程和过去会议纪要可在此 HackMD 中查阅。

日历#

此日历显示所有社区会议。我们的社区会议非常适合任何希望为 pandas 做出贡献，或只是好奇当前开发进展的人。

您可以使用以下链接订阅此日历

  • iCal

  • Google 日历

此外，我们有时还会就特定主题举行一次性会议。这些会议也将发布在此日历上。

GitHub 问题跟踪器#

pandas 贡献者社区主要通过此渠道进行交流。任何社区成员都可以通过提出问题来：

  • 报告错误，例如“我注意到某个函数的行为不正确”

  • 请求功能，例如“我希望这个错误消息更具可读性”

  • 请求文档改进，例如“我发现本节内容不清楚”

  • 提问，例如“我注意到某个函数的行为在不同版本之间发生了变化。这是预期的吗？”。

    理想情况下，您的问题应与 pandas 的工作方式而非如何使用 pandas 有关。StackOverflow 更适合回答使用方面的问题，我们要求所有使用方面的问题都首先在 StackOverflow 上提出。感谢您尊重我们的时间和意愿。🙇

维护者和经常性贡献者也可能提出问题来讨论项目的持续开发。例如：

  • 报告 CI、GitHub Actions 或 pandas 性能方面的问题

  • 提出与内部结构相关的问题

  • 启动路线图讨论，就未来版本中要做的事情或 API 更改的提案达成一致。

  • 提出与项目网站、徽标或治理相关的问题

开发者邮件列表#

pandas 邮件列表 pandas-dev@python.org 用于进行长篇对话，并吸引那些可能不活跃在问题跟踪器上，但我们希望将其纳入讨论的更广泛社区成员。

社区 Slack#

我们为贡献者、维护者和潜在贡献者提供了一个聊天平台。这里不是回答用户问题的地方，而是解答有关贡献 pandas 的问题。Slack 是一个私人空间，特别适合那些不愿在大型公共邮件列表或 GitHub 上提出问题或想法的人。

如果这听起来像是适合您的地方，欢迎通过此链接加入！请记住遵守我们的行为准则，并请注意，我们的管理员正在监控不相关的信息，并将移除利用我们的 Slack 进行垃圾邮件、广告和与 pandas 贡献社区无关消息的人员。此外，请记住 Slack 并非旨在取代邮件列表或问题跟踪器——所有重要的公告和对话仍应在那里进行。
